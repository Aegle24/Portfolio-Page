Container 01

The first set of containers we have made are these light blue boxes. The items of this container are the div, the h2 element, the paragraph and the element we have applied the feat-item class to. In this container, it’s the code snippet. In the second container, it’s the image. In order to let the browser know which element is the container, we use the container-type property and set it to inline-size.

There are three possible values we could have set container-type to:

Inline-size
Typically this is the width but since it uses the inline direction, for some elements, it could be the height.
Size
This uses both the inline and block direction which is the height and width.
Normal
This is used to query not the size but the style of the element. This is still experimental and not fully supported so it is advised to stay away from this value for now.

.feat {
	display: flex;
	flex-direction: column;
	container-type: inline-size;
	/*Other statements*/
}

.feat-item {
	width: 100%;
	min-width: 300px;
	margin: auto;
	padding: 20px;
}

@container (max-width: 700px) {
	.feat-item {
		min-width: 250px;
		padding: 10px;
		
		@include qmax($mobile) {
			padding: 0;
		}
	}
}

Container 02

We created a query so when the container is 700 pixels or less. It will set the min-width of the feat-item to 250px. We also add a padding of 10 pixels around it. When the container is larger than 700 pixels, the min-width of feat-item is 300 pixels.

Here are a couple things you need to keep in mind when working with container-queries:

  • You can’t change the characteristics of the container with a container query. In our code, we can change the properties of div, h2, paragraph, pre, and even code. But we can’t affect the section tag we applied the class feat to.
  • In order to use a container query, you need to assign at minimum the container-type property to an element. You can apply container-name as well but it is not required.
  • In order to use a container query, you need to assign at minimum the container-type property to an element. You can apply container-name as well but it is not required.
  • To apply the type and name on the same line use the syntax container: name/type;
  • With container queries came new units of measure. Some of these units are container width and height (cqw/cqh), container logical directions (cqi/cqb), container minimum and maximum lengths (cqmin/cqmax), ch, etc.
  • Container style queries are experimental and can only evaluate custom CSS properties right now.

Container 03

Below I have applied another container to the grid of cards. This time I named it cards so it only affects this container. I measure the container by how many characters wide it is and I use it to change the layout of the grid within the individual cards.

The Code:


@container cards (min-width: 60ch) {
	.card {
		display: grid;
		grid-template-columns: auto 1fr;
		
		h3 {
			margin-top: auto;
		}
		
		p {
			grid-column: 1 / 3;
		}
	}
}

@container cards (width > 200ch) {
	.card {
		display: block;
	}
}
